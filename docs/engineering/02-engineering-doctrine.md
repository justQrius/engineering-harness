# Engineering Doctrine

Use a lightweight system that preserves speed without losing rigor.

## Core Rules

1. Anchor meaningful work in project docs and real code.
2. Use external sources deliberately and prefer primary sources.
3. Use coding agents as specialized workers, not interchangeable chatbots.
4. Prefer thin vertical slices.
5. Use tests for deterministic behavior and evals for agent behavior.
6. Keep changes reviewable.
7. Record durable decisions.
8. Make risks visible.
