# Architecture Docs

Use this folder for chosen system structure and durable technical decisions.

Keep here:

- reference architecture
- subsystem architecture docs
- trust-boundary maps
- runtime topology or storage model docs
- architecture decision records (ADRs)

Use `docs/architecture/drafts/` for competing or not-yet-ratified proposals.

Do not keep here:

- external research scans; keep those in `research/`
- roadmap or sequencing plans; keep those in `docs/planning/`
- local feature or subsystem designs that are not yet canonical; keep those in
  `docs/design/`
- workflow doctrine; keep that in `docs/engineering/`
