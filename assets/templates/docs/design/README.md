# Design Docs

Use this folder for approved feature, subsystem, API, data-flow, and UX design
docs.

Keep here:

- feature design briefs
- subsystem design docs
- interface and API proposals
- data-flow or workflow designs
- UX or operator-flow specs when they affect implementation

Do not keep here:

- raw research and external scans; keep those in `research/`
- durable system-wide architecture; keep those in `docs/architecture/`
- process doctrine; keep those in `docs/engineering/`
- live packet state; keep that in `.handoff/`

Promote a design doc into `docs/architecture/` when it becomes a durable
system-wide decision rather than a local slice design.
