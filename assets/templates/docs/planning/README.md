# Planning Docs

Use this folder for durable planning artifacts that sit between vision and live
execution.

Keep here:

- phase maps and roadmaps
- initiative or program plans
- approved proposals that define sequencing or milestones

Do not keep here:

- external landscape scans or raw synthesis; keep those in `research/`
- chosen system structure or ADRs; keep those in `docs/architecture/`
- feature or subsystem design docs; keep those in `docs/design/`
- live execution state; keep that in `.handoff/`

Practical rule:

- if the document explains what should happen over time, it belongs here
- once a plan becomes active work, mirror the current slice in `.handoff/CURRENT.md`
